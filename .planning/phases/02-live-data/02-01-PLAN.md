    2. **`lib/txline/txLineSseSource.ts`** (per D-03 override, D-04, D-05, RESEARCH.md Sec "Pattern 1" + "Code Examples — TxLineSseSource Implementation Pattern"):
    - Import types: `TxLineSource` from `./TxLineSource`, `TxLineEventPacket` from `@/lib/schema/txLineSchema`, `ConnectionState`/`ConnectionEventBus` from `./txLineConnectionBus`
    - Import adapters: `readSseMessages`, `parseScoreEvent`, `parseOddsEvent`, `mergeScoresAndOdds` from `./txLineSseAdapter`
    - Import `DEVNET_ORIGIN, SSE_SCORES_PATH, SSE_ODDS_PATH, STALE_THRESHOLD_MS, MAX_RETRIES, STALE_CHECK_INTERVAL_MS` from `./txLineFixtureIds`
    - `SseSourceConfig` interface: `{ jwt, apiToken, fixtureId, onConnectionChange? }`
    - `createTxLineSseSource(config: SseSourceConfig): TxLineSource & { connectionBus: ConnectionEventBus }`:
      - `connectionBus = new ConnectionEventBus()`
      - `subscribers = new Set<(packet: TxLineEventPacket) => void>()`
      - `mergedState: Partial<TxLineEventPacket> = {}`
      - `lastPacketTime = 0`, `abortController: AbortController | null = null`
      
      Internal `async function connectStream(url, parser)`:
      - Use `fetch(url, { headers: { Authorization, X-Api-Token, Accept, Cache-Control }, signal })` — **NOT native EventSource** per D-03 override
      - On non-ok response, throw with status code
      - Iterate `for await (const message of readSseMessages(response))` 
      - For each message: `JSON.parse(message.data)` → call parser → merge into mergedState → emit via subscribers
      - Update lastPacketTime
      
      Internal `async function connectWithRetry(maxRetries = MAX_RETRIES)`:
      - For attempt 0..maxRetries:
        - connectionBus.setState("connecting" or "reconnecting")
        - abortController = new AbortController()
        - `Promise.all([connectStream(scoresUrl, parseScoreEvent), connectStream(oddsUrl, parseOddsEvent)])` — both streams in parallel per D-04
        - On success: connectionBus.setState("connected"), return
        - On error: if aborted, break; else exponential backoff `min(1000 * 2^attempt + random*500, 8000)`
      - After all retries: connectionBus.setState("disconnected")
      
      Return object: `{ subscribe(callback) }` — implements TxLineSource:
      - Add callback to subscriber set
      - If first subscriber: call `connectWithRetry()`, start stale detection interval (`setInterval` checking `Date.now() - lastPacketTime > STALE_THRESHOLD_MS`, sets state to "stale")
      - Return unsubscribe function that removes callback from set; if no subscribers left, abort controller + set state disconnected

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → TxLINE API | SSE connection crosses from browser to devnet API over HTTPS. Custom headers (Authorization, X-Api-Token) transmitted. |
| SSE stream → React state | Raw SSE JSON enters adapter before reaching React. Zod validation at mergeScoresAndOdds() guards against malformed payloads. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Information Disclosure | SSE connection (Authorization header) | high | mitigate | JWT/API token stored in-memory only; never persisted to localStorage, IndexedDB, or SW cache. Token re-fetched on 401 (per RESEARCH.md Pitfall 4). |
| T-02-02 | Tampering | txLineSseAdapter.mergeScoresAndOdds | high | mitigate | Zod schema validation (txLineEventPacketSchema.parse()) rejects malformed or incomplete packets before they reach React state. |
| T-02-03 | Information Disclosure | Client bundle | medium | mitigate | JWT/API token obtained at runtime via TxLINE API calls, never bundled as build-time constants. |
| T-02-04 | Denial of Service | SSE retry loop | low | accept | Exponential backoff caps at 8s max with 5 retries. App falls back to API_ERROR state gracefully. |
| T-02-SC | Tampering | npm package installs | high | mitigate | serwist and @serwist/turbopack both [ASSUMED] per RESEARCH.md Package Legitimacy Audit — no blocking checkpoint needed. |
</threat_model>
